v0.6 - The Honeymoon Massacre

138K → 81K uncompressed / 37K packed

  • Binary size cut by 40% (73% with UPX LZMA). 81K raw, 37K packed.
  • Zero heap allocations in the render loop. Replaced std::string output_buffer with a char[4096] stack buffer. The frame is built entirely on the stack, flushed in one write() syscall.
  • Killed <fstream> entirely. All file I/O (4 files: buffer, config, keybinds, history) replaced with raw POSIX open/read/write/close. No more basic_filebuf vtables.
  • std::map + std::function → enum + switch. Action dispatch is now a flat array lookup and a jump table. No red-black tree, no type-erased function wrappers.
  • std::shared_ptr<KeyNode> → raw linked list. The key trie uses raw pointers. No atomic reference counting on every keystroke.
  • <ranges> → manual loops. std::views::split / drop / take replaced with index arithmetic. 6K of <ranges> template instantiations removed from .text.
  • std::string clipboard → char* + malloc. No SSO, no std::string copy/move overhead.
  • std::vector<Key> → Key[16] + counter. Max keychord depth is 4. Why was this a vector.
  • Vectors → constexpr C arrays. Menus, color tables, action names all baked into .rodata at compile time.
  • std::stringstream → memchr + while loops. Config and keybind parsing now runs on raw char* buffers.
  • <iostream> → write() syscalls. No locale init at startup. No static initializers.
  • std::stoi + try/catch → strtol. No exception tables needed for parsing integers.
  • Removed unused config fields: color_theme, scroll_offset, font_size (stored but never used) eliminated from the Config struct.

Flags used: -fno-exceptions, -fno-plt, -fno-rtti, -fno-unwind-tables, -fmerge-all-constants, -fno-unroll-loops, --icf=all, --hash-style=sysv, --gc-sections.

Sections eviscerated: .plt (624→20 bytes), .got.plt (328→20 bytes), .rela.plt (912→0 bytes), .gcc_except_table (3.4K→0 bytes), .eh_frame (5.5K→48 bytes).

v0.5 - Config Mixin, Undo/Redo & Working Settings

commit 6815087

  • Extracted Config from a nested struct into a proper CRTP-compatible mixin (Editor : private Config, private UndoHistory<>)
  • Settings persist to ~/.config/honeymoon/config.moon with XDG-aware path resolution and automatic directory creation
  • Full key set (tab_width, show_line_numbers, syntax_highlighting, color_theme, scroll_offset, font_size) serializes as clean key value pairs with # comment support
  • UndoHistory<> mixin brings snapshot-based undo/redo with contiguous-typing coalescing (typing gate so hello is one undo step)
  • Every buffer-mutating path snapshots before touching state (cut, yank, kill_line, kill_word, transpose_chars/words, indent/dedent, delete_forward/backward, newline)
  • Undo/redo applies via delete_range(0, size()) + insert_string (O(n) restore, O(1) snapshot)
  • Bound C-/ → undo, M-/ → redo
  • Settings screen actually works: arrow keys navigate, Enter toggles booleans, Tab Width cycles 2→4→8, and every change auto-saves
